 1.3.1 (released Feb 09, 2010)
Introduction
TideBrowse 1.3.1 (and the whole TIDE-Suite) was updated to JEdit 4.3.1. and the most recent 3rd party plugins, especially the current ProjectViewer 2.9.3.
Features
  • Updated to use the most current 3rd party libs (Project Viewer, Console, etc.)
  • TIDEDebug changed its idea of "patching" main.cs - the file is only temporarily modified on startup by adding a single line script command.
 1.3.0 (released May 27, 2009)
Introduction
TideBrowse 1.3.0 (and the whole TIDE-Suite) is compatible to JEdit 4.3.
Features
  • TIDE now supports JEdit 4.3
  • TIDEBrowse features an additional "Search current selected function in project" function
 0.0.4 (released Nove. 27, 2006)
Introduction
TideBrowse 0.0.4 (and the whole TIDE-Suite) is compatible to JEdit 4.2.
Features
  • TIDEBrowse now features simple "autocomplete-as-you-type", which can be toggled in the Options Dialog. It doesn't support fancy context-sensitive, object based function- ect. completions, but it's quite handy imho. For general completions, the file "autocomplete.txt" in the JEdit home dir is used. In addition to that, a file named "project_autocomplete.txt" is generated and saved for every opened TIDE-project (for .cs and .gui files) the first time the project is opened. You can re-generate this file manually from the Plugins-TIDE-TIDEBrowse menu if your project has changed or if you've added a lot of stuff to your files which you'd like to have in the autocompletion.
  • I've also fixed some bugs, e.g. the one that would stop TIDEBrowse from parsing a file when the term "datablock" (as opposed to "dataBlock") was used inside a function.
 0.0.3 (released Dec. 12, 2003)
Introduction
TideBrowse 0.0.3 does not require the modified "ProjectViewer" plugin anymore. It is now compatible with JEdit 4.1 final (and up, hopefully) and uses the standard ProjectViewer plugin (current version I've tested with is 2.0.2)
Features
  • Fixed incompatibility with JEdit versions > 4.0.x (caused by ProjectViewer version conflicts). You can now use the standard ProjectViewer plugin (>= 2.0.2). Have fun!
  • TIDEBrowse now uses the file types from the ProjectViewer configuration for the "Search for selection..." search - select any text in the current buffer/file, hit the Search button and it will search the current project using the valid file types / extensions of ProjectViewer (don't forget to add .mis, .cs, .gui, .hfl, .dml, .log, .cc, .h, ... there!)
 0.0.2 (released July. 19, 2002)
Introduction
TideBrowse 0.0.2 requires a modified "ProjectViewer" plugin to be installed.
Features
  • New "Search current selection in project" function - requires modified "ProjectViewer" plugin which is also included in the TideBrowe zip you've downloaded.
 0.0.1 (released July. 17, 2002)
Introduction
TideBrowse 0.0.1 is the first TideBrowse version.
It is based on the JBrowse plugin written by George Latkiewicz and Andre Kaplan.
Features
  • Works with jEdit > 4.0pre4
