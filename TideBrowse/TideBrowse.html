Introduction

The "TideBrowse" plug-in provides a tool for navigating through Torque script code. It is a stripped-down version of the JBrowse plugin written by George Latkiewicz and Andre Kaplan changed to handle *cs, *.gui and *.mis files instead of *.java files.
And instead of listing all classes, packages, attributes, etc. of Java files it now lists all Torque functions, datablocks and objects.
It displays a navigation window which uses a tree view to display the hierarchy of functions, datablocks and object instantiations for the file in the current buffer. A single mouse click on one of the nodes in the tree will highlight the line in the jEdit buffer where the definition of that item begins. Options are provided to control both what is displayed and how.
Another feature is the "Search current selection in project" (accessible through the "Plugins->TideBrowse" menu or via the "Search" button in the TIDEBrowse gui): you can select any word/phrase in the current file and search for it in the entire project directory (you can include your engine source folders, too) - it searches in all the file types you've entered for the ProjectViewer ("Utilities -> Global Options -> Plugins -> ProjectViewer"). Don't forget to add Torque related extensions there, e.g.

  • *.cs, *.gui
  • *.cc, *.h
  • *.mis, *.dml
  • *.hfl
  • *.hlsl
  • *.log

This version requires:

  • JDK >= 1.4
  • jEdit >= 4.3.x
  • "ProjectViewer" plugin >= 2.1.3.7 (for "Search current selection in project" function etc.)
  • SideKickPlugin 0.1 (for Autocompletion)
  • ErrorListPlugin 1.2 (for Autocompletion)

For the TideBrowse revision history, have a look at the Change History.

Understanding the TideBrowse Tree Display

TideBrowse displays the hierarchy of elements in a tree using the options (see below) you have specified. Each element is displayed with one of the following icons:

 Function - all torque script functions.

 Datablock - all Torque datablocks.

 Object - new objects (e.g. Terrain, Sun, TSStatic, Trigger,...).

 Error - indicates the location of a point in the code where TideBrowse encountered what it considered a parse error. If any such errors were found an indicator appears next to the resize button. You may need to fix the error and re-parse to obtain a full hierarchy for the buffer. Any code for which TideBrowse reports a parse error should not compile without generating an error as well (if it does then TideBrowse has a bug, please let me know!).

TideBrowse Controls

Parse button - Clicking this button will cause the tree hierarchy to be re-built based on the current contents of the active buffer. When TideBrowse is opened, it automatically does this (unless the file does not have a TGE related extension like .cs or .gui). Changing any options will automatically update the tree display. Thus, the only reason to re-parse is because the file contents have changed, or a different buffer has been activated or because you are curious what TideBrowse will do if you attempt to parse a binary file.

Configure button - Clicking this button will open, or bring to the foreground, the TideBrowse options dialog. This dialog allows you to change, on-the-fly, the current option settings for this TideBrowse session. More information on this dialog, including the effect of each option setting is described below in the TideBrowse Options section.

Find button - Clicking this button will search the currently active project for the current selection in the active file/buffer and display the results in the HyperSearch Results pane which will open as you click. It uses the file extensions configured for the ProjectViewer plugin.

Help button - Clicking this button will open the Help Viewer with the documentation for TideBrowse (the HTML page you are now reading).

Navigating the TideBrowse Tree

The tree can be navigated using both the mouse and the keyboard. The keys and their associated actions, as listed below, are those that are common to all the standard look & feels.

Action Key
Expand/collapse node Right, Left
Move up/down one node Up, Down
Move to first/last node Home, End
Move up/down one page PgUp, PgDn
TideBrowse Options

TideBrowse is highly configurable, using a set of option values that control the behavior and appearance of the tree that is displayed for a parsed Torque script file. A set of default option values is stored in a property file, which is loaded each time a new TideBrowse session is started. While TideBrowse is running these options may be changed on-the-fly using the TideBrowse Options Dialog. The current option values can be saved as the new defaults by selecting the "Set As Defaults" button and the last saved defaults can be loaded to replace the current values by selecting the "Restore Defaults" button. The default option values may also be set using the TideBrowse tab of the jEdit plug-in options dialog (accessible from the jEdit menu item Utilities->Global Options).

The tables below describe the effect of each option value. These options are grouped into three basic categories:

General options:

Display status bar  If set then the status bar, which indicates the total number of classes, interfaces, attributes and methods in the currently parsed file, will be displayed at the bottom of the TideBrowse window.
Automatic reparse If set, TideBrowse automatically reparses the buffer and updates its display when the jEdit buffer changes.
Sort If set, TideBrowse sorts the different elements by their name.

Display options (How to display):

These options specify the format that is to be used to display the elements in the tree. Changing any of these options will normally update the display without reloading the tree.
Note: if the tree currently has a very large number of displayed nodes (more than 200) then it will be reloaded rather than updated for performance reasons.

General Display Options

The following options can be set independently of the current Display Style.

Keywords specified by icons If set then the keywords directly associated with the icons for functions, datablocks & objects will be displayed, otherwise they are suppressed.
Line Numbers If set then the line number at which the definition of each element begins will be displayed.
